## Filtering Criteria

1. **Compute:** Medium — 8-16x NVIDIA H200 GPUs (~$50K-$150K per experiment)
2. **Scope:** LLM or agent related. Must have theoretical, model architecture, loss function, or training methodology contributions. Exclude pure CV/robotics/physics/benchmark-only.
3. **Maturity:** Not fully mature; room for follow-up work at top venues (ICLR, NeurIPS, ICML, ACL, EMNLP)
4. **Novelty:** Prefer new frameworks, theoretical connections, paradigm shifts over incremental improvements

### 1. Attention Residuals

**Paper #37 — ArXiv [2603.15031](https://arxiv.org/abs/2603.15031) — 2026-03-19**

**490 likes.** Residual connections with PreNorm are standard in modern LLMs, yet they accumulate all layer outputs with fixed unit weights. This paper shows this uniform aggregation causes uncontrolled hidden-state growth with depth, and proposes learned residual weights to fix it. A simple but potentially transformative change to how all LLMs are built.

### 2. Thinking Deeper, Not Longer: Depth-Recurrent Transformers for Compositional Generalization

**Paper #83 — ArXiv [2603.21676](https://arxiv.org/abs/2603.21676) — Authors at cs.LG, cs.CL, cs.AI — 2026-03-23**

Proposes "vertical Chain-of-Thought": instead of generating more tokens horizontally, the model "thinks deeper" by iterating through depth-recurrent transformer blocks. A fixed perception interface encodes input, then a recurrent reasoning core iterates variable depth. Demonstrates compositional generalization on graph reachability, nested logic, and LEGO tasks.

| Field | Details |
|---|---|
| **Topic Category** | LLM Architecture / Reasoning / Depth-Recurrence |
| **Importance** | 5/5 — Orthogonal paradigm to horizontal CoT. If it scales, could replace token-intensive reasoning |
| **Possible Directions** | (a) Scale to full LLM pretraining (current experiments are small); (b) Combine with horizontal CoT for hybrid reasoning; (c) Adaptive depth selection (stop iterating when confident); (d) Apply to mathematical theorem proving; (e) Theoretical analysis of expressivity vs. depth |
| **Compute Estimate** | 4-8x H200 for proof-of-concept, 16x H200 for scaling experiments |
| **Data Estimate** | 10-100B tokens + synthetic reasoning tasks |
| **Datasets** | Graph reachability, nested logic, LEGO, potentially GSM8K/MATH for scaling |
| **Top-Venue Probability** | **80%** — Novel paradigm, but needs scaling validation. Strong ICML/NeurIPS fit |

**Paper #7 — ArXiv [2603.22117](https://arxiv.org/abs/2603.22117) — 2026-03-23**

## Top 5 Recommendations (Ranked)

### 1. Attention Residuals (Paper #37) — **85%**
By far the highest community signal (490 likes). A simple architectural change (learned residual weights vs. fixed unit weights) that could become standard in all future LLMs. Highly replicable at medium compute, clear follow-up directions including interaction with MoE routing and different normalization schemes. The simplicity is its strength — easy to validate and extend.

### 2. Thinking Deeper, Not Longer (Paper #83) — **80%**
A genuine paradigm shift: vertical CoT via depth-recurrent transformers. Instead of generating more tokens, iterate through depth. Orthogonal to all existing CoT work. Current experiments are small-scale (graphs, logic, LEGO), creating a huge opportunity to be the first to scale this to full LLM reasoning. High-risk, high-reward.

### 3. On the Direction of RLVR Updates (Paper #7) — **75%**
Mechanistic understanding of RLVR at the token level. Identifies that updates are sparse and directional, with gradient magnitude tied to token probability. Directly actionable: the exploitation strategy boosts reasoning. Follow-ups could design better RL algorithms informed by this analysis.

### 4. Holistic MoE Scaling Laws (Paper #71) — **80%**
Fills a critical gap: how MoE architectural parameters interact under scaling. Practical impact for anyone designing MoE models. The interaction between expert count, expert size, and top-k routing has not been systematically studied before. Strong NeurIPS/ICML candidate.

### 5. The Y-Combinator for LLMs (Paper #22) — **70%**
The most creative paper in this batch. Bridges programming language theory (lambda calculus) with LLM context management. The theoretical contribution is elegant; the empirical validation needs strengthening. High-novelty, moderate-risk.
